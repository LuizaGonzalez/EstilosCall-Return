## General Description

This workshop introduces the fundamental concepts of network communication using Java. It covers the **TCP** and **UDP** protocols, working with URLs, client-server sockets, datagrams, and finally remote method invocation with **RMI**.

Throughout the workshop, clients and servers are implemented in progressively more complex ways, ranging from simple URL reads to a distributed chat with remote objects.

EXERCISE 3
---
### Objective
Implement a TCP client-server pair where the client sends an integer and the server responds with its square.

### What is done?
- **Server:** opens a **ServerSocket** on port 35000, accepts a connection, reads the received number, calculates the square and returns it as text.
- **Client:** creates a **Socket** to **127.0.0.1:35000**, reads a number from the console, sends it and displays the response.
### How to run
1. Compile the project.
2. Launch the server first; it will wait for connections.
3. Launch the client in another terminal, enter a number and observe the response.
   First we run the server, which stays waiting, then we run the client. The client prompts for a number and the server returns its square.

### Objective
Extend the TCP server to compute sine, cosine, or tangent depending on the active function. The function can be changed at runtime by sending a message with the prefix **fun:** (e.g. **fun:sin**, **fun:cos**, **fun:tan**). By default it starts computing **cosine**.

### What is done?
The server maintains an internal state (**funcionActual**) that changes when it receives a message starting with **fun:**. If it receives a number, it applies the active function and returns the result. The client can mix function changes and numeric queries in the same session.

Implement an HTTP server capable of handling multiple consecutive (non-concurrent) requests and returning static files: HTML pages and images.

### What is done?
The server listens on port 6565. ## EXERCISE 6
 
---

### Objective
Use the UDP protocol to implement a client that queries the current time from a server every 5 seconds. If the server does not respond, the client retains the last received time and automatically resynchronizes once the server becomes available again.

### What is done?
What is done?

- **Server (ServidorHora):** listens on port 9696. Upon receiving any datagram, it responds with the current time as a String using new Date().toString().
- **Client (ClienteHora):** sends a datagram in a continuous loop and displays the time from the response. It uses setSoTimeout(2000) on the socket so that if the server does not reply within 2 seconds, a SocketTimeoutException is caught and the last known time is shown instead

Este taller introduce los conceptos fundamentales de comunicación en red usando Java. Se abordan
los protocolos **TCP** y **UDP**, el trabajo con URLs, sockets cliente-servidor, datagramas y,

pasando de lecturas simples de URLs hasta un chat distribuido con objetos remotos.

### ¿Qué se hace?

¿Qué se hace?
